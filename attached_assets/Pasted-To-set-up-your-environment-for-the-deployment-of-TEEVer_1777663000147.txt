To set up your environment for the deployment of TEEVerifier.sol and AssetRegistry.sol using Foundry, follow these steps to install the toolkit locally and configure your environment variables.
## 1. Install Foundry Locally
Foundry is a fast smart contract development toolkit written in Rust. You can install it on macOS, Linux, or Windows (via WSL) using the following commands:

* Download the installer: Run this command in your terminal:
curl -L https://foundry.paradigm.xyz | bash
* Install the binaries: After the installer completes, restart your terminal or run source ~/.bashrc (or the equivalent for your shell). Then, run:
foundryup
* Verify installation: Confirm everything is working by checking the version of Forge:
forge --version [1, 2, 3] 

This will install the full suite, including Forge (testing/deployment), Cast (chain interaction), and Anvil (local node). [4, 5] 
------------------------------
## 2. Configure Environment Variables
Instead of typing sensitive keys directly into your terminal, it is best practice to use a .env file in your project's root directory.

   1. Create a .env file: In your project folder, create a file named .env and add the variables mentioned in your image:
   
   PRIVATE_KEY=your_wallet_private_key
   ORACLE_ADDRESS=0x...
   ENCLAVE_ADDRESS=0x...
   POLYGON_AMOY_RPC_URL=https://polygon.technology
   ```
   
   2. Load the variables: Before running your deployment script, load these into your current terminal session:
   source .env
   3. Security Tip: Ensure your .env file is added to your .gitignore to prevent accidentally pushing your private key to GitHub. [6, 7, 8, 9] 

------------------------------
## 3. Run the Deployment Test
Once Foundry is installed and variables are loaded, you can execute the command from your image:

forge script script/DeployFull.s.sol --rpc-url $POLYGON_AMOY_RPC_URL --broadcast

If you just want to test without sending a real transaction, add the --dry-run flag as suggested in your screenshot.
