# AI SDK + Vercel Sandbox Setup

## Overview

This document describes the AI SDK infrastructure configured for ProofBridge Liner:
- **Code execution**: Vercel Sandbox runs arbitrary Python code in isolated Firecracker microVMs
- **AI model access**: NVIDIA NIM API (OpenAI-compatible) for LLM orchestration
- **Tool integration**: `ai-sdk-tool-code-execution` bridges LLM → sandbox code execution

## Package Stack

| Package | Version | Purpose |
|---------|---------|---------|
| `ai` | ^6.0.189 | AI SDK Core — `generateText`, `generateObject`, streaming |
| `ai-sdk-tool-code-execution` | ^0.0.2 | `executeCode()` tool — sandboxed Python execution |
| `zod` | ^4.4.3 | Schema validation for tool parameters |
| `@ai-sdk/openai` | ^3.0.64 | OpenAI provider (NVIDIA NIM backup) |
| `@ai-sdk/openai-compatible` | ^2.0.47 | Command Code API provider (primary) |

## AI Provider — Command Code API

- **Endpoint**: `https://api.commandcode.ai/provider/v1` (OpenAI-compatible)
- **Vercel env var**: `COMMAND_CODE_KEY` (set in Preview + Production)
- **Local env var**: `CMD_API_KEY` (alias in `.env.local`)
- **Models**: `deepseek/deepseek-v4-flash`, `claude-sonnet-4-6`, `claude-opus-4-7`, `gpt-5.4-mini`, `google/gemini-3.5-flash`

```typescript
import { createOpenAICompatible } from '@ai-sdk/openai-compatible'

const cmd = createOpenAICompatible({
  name: 'command-code',
  apiKey: process.env.CMD_API_KEY,
  baseURL: 'https://api.commandcode.ai/provider/v1',
})

const result = await generateText({
  model: cmd.chatModel('deepseek/deepseek-v4-flash'),
  tools: { executeCode: executeCode() },
  maxSteps: 5,
  prompt: 'Calculate and return the 10th Fibonacci number.',
})
```

### Vercel Sandbox

- **Auth**: `VERCEL_OIDC_TOKEN` (OIDC token, auto-generated via `vercel link` + `vercel env pull`)
- **Runtime**: Python 3.13, Node.js 24/22/26, Amazon Linux 2023
- **Isolation**: Firecracker microVM per sandbox
- **Verified**: `sum(1..100)` returned `5050` with exit code 0

### Environment Variables

Set in `.env.local` (gitignored, pulled via `vercel env pull`):

```bash
VERCEL_OIDC_TOKEN=<auto-generated JWT>
OPENAI_API_KEY=<NVIDIA NIM API key (nvapi-...)>
```

## Usage

### Basic code execution (no LLM)

```typescript
import { executeCode } from 'ai-sdk-tool-code-execution'

const tool = executeCode()
const result = await tool.execute(
  { code: 'print(sum(i for i in range(1, 101)))' },
  { toolCallId: 'test-001' }
)
// { output: "5050\n", exitCode: 0, stderr: "" }
```

### With an LLM (AI SDK)

```typescript
import { executeCode } from 'ai-sdk-tool-code-execution'
import { generateText } from 'ai'
import { createOpenAI } from '@ai-sdk/openai'

const provider = createOpenAI({
  apiKey: process.env.OPENAI_API_KEY,
  baseURL: 'https://integrate.api.nvidia.com/v1',
  compatibility: 'compatible',
})

// Note: NVIDIA NIM function calling support varies by model.
// Tested models: deepseek-ai/deepseek-v4-flash (selects tool but no args),
//                meta/llama-4-maverick-17b-128e-instruct (returns JSON inline)
const result = await generateText({
  model: provider.chatModel('meta/llama-4-maverick-17b-128e-instruct'),
  tools: { executeCode: executeCode() },
  maxSteps: 5,
  prompt: '...',
})
```

## Model Compatibility (NVIDIA NIM)

| Model | Tool Selection | Args Population | Notes |
|-------|---------------|-----------------|-------|
| `deepseek-ai/deepseek-v4-flash` | ✅ Yes | ❌ No | Selects tool, empty args |
| `qwen/qwen3-coder-480b-a35b-instruct` | ✅ Yes | ❌ No | Selects tool, empty args |
| `nvidia/llama-3.3-nemotron-super-49b-v1` | ✅ Yes | ❌ No | Selects tool, empty args |
| `meta/llama-4-maverick-17b-128e-instruct` | ❌ No (inline) | N/A | Returns function JSON as text |

Models on `integrate.api.nvidia.com/v1` have inconsistent OpenAI-compatible function calling support. For reliable tool use, use OpenAI or Anthropic directly.

## Vercel Deployment Guard

The `scripts/replit-vercel-prod-deploy.sh` script gates production deploys:

```
[Git Branch/Commit] → [Clean Worktree] → [Vercel Token] → [Route Guard]
→ [Lockfile] → [Build/Tests] → [Vercel CLI] → [Alias Guard]
→ [Env Pull] → [Contract Address Guard] → [Build] → [Deploy] → [Health Check]
```

Run with:
```bash
export VERCEL_AUTH_TOKEN=...
export EXPECTED_GIT_BRANCH=gate-1
export EXPECTED_GIT_HEAD=<commit>
export CONFIRM_PROD_DEPLOY=yes
npm run deploy:vercel:replit
```
