Gate-1 Semantics

Every contract invariant, explicitly stated

Intake

deed_hash intake

Every request carries a 64-hex SHA-256 deed hash. No request without a deed hash passes Gate-1.

Computation

posterior computation

Beta-Binomial posterior: μ = (α+1)/(α+β+2), calibrated threshold: τ = τ₀/(1+γ·β/α). Active amend chains form sealed interfaces around chain ref and envelope ref at the time of test.

Decision

PASS | WARN | HALT

μ > τ → PASS. μ = τ → WARN. μ < τ → HALT. Three-way verdict. SAFE / TRIP replaced across the Gate-1 surface.

Receipt

receipt_id · server-generated

UUID v4 generated by the server at response time. Never provided by the client as part of the request body.

Pipeline

pipeline_hash · server-generated

SHA-256 over the canonical handshake + envelope JSON at the time of test. Chain and envelope references are included.

Anchoring

anchored_at: null

Active amend chains do not overwrite the anchored_at field. Once null, always null at Gate-1. Enforced by the server handler.

Signature

signed response payload

HMAC-SHA256 over `posterior:threshold:verdict`. Key held in `PROOFBRIDGE_HMAC_SECRET`. Never returned in the body.

Signal

safegrid_signal

Response block with evaluator_version, verdict, and server-generated signal_id. Verdict matches the top-level verdict exactly.
